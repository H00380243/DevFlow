## 4. Risk-Based Prioritization

1. **Critical path**: Webhook → parse → review → design → implement → deliver (FR-001→FR-017b)
2. **Integration boundaries**: State machine transitions across stages (FR-020, FR-021)
3. **NFR thresholds**: Security auth + audit (NFR-006, NFR-007, NFR-008)
4. **Edge cases**: Error recovery, max retries, timeouts (FR-012, FR-015)
5. **Compatibility**: Browser rendering of 3 UI pages (NFR-011)
